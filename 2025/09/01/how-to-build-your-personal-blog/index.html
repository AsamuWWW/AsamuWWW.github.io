从0开始搭建个人独立博客：工具安装篇

Building a Personal Independent Blog from Scratch: Tool Installation Guide

这篇文章将详细分享个人独立博客搭建的第一步——工具准备与安装，全程无废话、不跳步，适配 Windows 11（64位）系统，跟着操作即可完成基础环境搭建。

This article will detail the first step in building a personal independent blog — tool preparation and installation, with no unnecessary content or skipped steps. It’s compatible with Windows 11 (64-bit), and you can complete the basic environment setup by following the instructions.

一、准备工具清单

1. List of Required Tools

搭建博客需先安装两款核心工具：Node.js（博客框架运行环境）和 Git（代码版本控制与部署工具），具体版本及作用如下：

To build a blog, you first need to install two core tools: Node.js (the runtime environment for the blog framework) and Git (the tool for code version control and deployment). Their specific versions and functions are as follows:

工具名称 作用说明 适配版本（Windows 11 64位） 下载地址
Node.js 基于 Chrome V8 引擎的 JavaScript 运行时，为 Hexo 博客框架提供运行环境 node-v24.6.0-x64 https://nodejs.org/en/
Git 分布式版本控制系统，用于后续博客代码的管理与部署（如部署到 GitHub Pages） 2.51.0（x64 version） https://git-scm.com/downloads

二、Node.js 安装步骤（Windows 11 64位）

2. Node.js Installation Steps (Windows 11 64-bit)

Node.js 是 Hexo 框架的核心依赖，必须优先安装，以下是完整安装流程：
Node.js is a core dependency of the Hexo framework and must be installed first. The complete installation process is as follows:

1. 下载 Node.js

1. Download Node.js

  1. 打开 Node.js 官网：https://nodejs.org/en/；

  2. Open the official Node.js website: [https://nodejs.org/en/];

  3. 点击首页显著位置的 “Get Node.js” 按钮（默认下载最新稳定版，本文以 node-v24.6.0-x64 为例）。

  4. Click the prominent “Get Node.js” button on the homepage (it downloads the latest stable version by default, and this article uses node-v24.6.0-x64 as an example).

2. 开始安装 Node.js

2. Start Installing Node.js

  1. 找到下载完成的安装包 node-v24.6.0-x64.msi，双击运行；

  2. Locate the downloaded installation package node-v24.6.0-x64.msi and double-click to run it;

  3. 弹出安装向导后，点击 【Next】 按钮；

  4. After the installation wizard pops up, click the 【Next】 button;

  5. 阅读许可协议，勾选 “I accept the terms in the License Agreement”（同意协议），点击 【Next】；

  6. Read the license agreement, check “I accept the terms in the License Agreement” (agree to the agreement), and click 【Next】;

  7. 选择安装路径：默认路径为 C:\Program Files\nodejs\，若需修改（如避免C盘占用过多），点击 【Browse】 选择自定义文件夹，确认后点击 【Next】；

  8. Select the installation path: The default path is C:\Program Files\nodejs\. If you need to modify it (e.g., to avoid occupying too much space on the C drive), click 【Browse】 to select a custom folder, confirm it, and then click 【Next】;

  9. 功能配置页：勾选 “Automatically install the necessary tools…”（自动安装必要工具，避免后续依赖缺失），点击 【Next】；

  10. Function configuration page: Check “Automatically install the necessary tools…” (automatically install necessary tools to avoid missing dependencies later), and click 【Next】;

  11. 等待安装进度完成，最后点击 【Finish】 按钮；

  12. Wait for the installation progress to complete, and finally click the 【Finish】 button;

  13. 安装完成后会自动弹出命令行窗口，按两次任意键，随后再 键入 ENTER，至此 Node.js 安装全部完成。

  14. After the installation is complete, a command prompt window will pop up automatically. Press any key twice, then type ENTER. At this point, the Node.js installation is fully completed.

三、Git 安装步骤（Windows 11 64位）

3. Git Installation Steps (Windows 11 64-bit)

Git 用于后续博客代码的提交与部署（如推送到 GitHub），以下是详细安装与配置流程：
Git is used for submitting and deploying blog code later (e.g., pushing to GitHub). The detailed installation and configuration process is as follows:

1. 下载 Git

1. Download Git

  1. 打开 Git 官网下载页：https://git-scm.com/downloads；

  2. Open the Git official website download page: [https://git-scm.com/downloads];

  3. 官网会自动识别系统，点击 “Windows” 对应的下载链接（本文以 2.51.0 x64 version 为例）。

  4. The official website will automatically recognize the system; click the download link corresponding to “Windows” (this article uses 2.51.0 x64 version as an example).

2. 开始安装 Git

2. Start Installing Git

  1. 找到下载完成的 Git 安装包（如 Git-2.51.0-64-bit.exe），双击运行；

  2. Locate the downloaded Git installation package (e.g., Git-2.51.0-64-bit.exe) and double-click to run it;

  3. 弹出安装向导，点击 【Next】 进入路径选择页：可自定义安装路径（如 D:\Program Files\Git），或保持默认，选择后点击 【Next】；

  4. After the installation wizard pops up, click 【Next】 to enter the path selection page: You can customize the installation path (e.g., D:\Program Files\Git) or keep the default. After selection, click 【Next】;

  5. 组件选择页（建议默认配置，无需修改）：绿色框是是否在桌面创建 Git 快捷方式（按需勾选），红框是默认勾选的核心组件（如“Git Bash”“Git GUI”，不建议修改），蓝色框是是否检查 Git 更新、向 Windows 终端添加 Git Bash 配置（按需勾选），确认后点击 【Next】；

  6. Component selection page (it’s recommended to keep the default configuration without modification): The green box is for whether to create a Git shortcut on the desktop (check as needed), the red box is for core components checked by default (e.g., “Git Bash”, “Git GUI”, modification not recommended), and the blue box is for whether to check for Git updates and add Git Bash configuration to the Windows Terminal (check as needed). Confirm and click 【Next】;

  7. 开始菜单文件夹命名页：默认“Git”即可，无需修改，点击 【Next】；

  8. Start menu folder naming page: The default “Git” is fine, no modification needed. Click 【Next】;

  9. 编辑器选择页：默认“Vim”（Git 自带编辑器，极少用到），直接点击 【Next】；

  10. Editor selection page: The default “Vim” (a built-in editor of Git, rarely used) is fine. Click 【Next】 directly;

  11. 分支名配置页：选择 “Let Git decide”（让 Git 使用默认分支名，新手友好），点击 【Next】；

  12. Branch name configuration page: Select “Let Git decide” (allow Git to use the default branch name, friendly for beginners) and click 【Next】;

  13. PATH 环境配置页（关键步骤）：推荐选择 “Git from the command line and also from 3rd-party software”（从命令行及第三方软件调用 Git，兼容性更强），点击 【Next】；

  14. PATH environment configuration page (key step): It’s recommended to select “Git from the command line and also from 3rd-party software” (call Git from the command line and third-party software for better compatibility) and click 【Next】;

  15. SSH 可执行文件选择页：选择默认选项 “Use bundled OpenSSH”（Git 自带的 OpenSSH 更稳定），点击 【Next】；

  16. SSH executable selection page: Select the default option “Use bundled OpenSSH” (the OpenSSH bundled with Git is more stable) and click 【Next】;

  17. HTTPS 后端传输选择页：选择 “Use the native Windows Secure Channel library”（使用原生 Windows 安全通道库，适配 Windows 系统），点击 【Next】；

  18. HTTPS backend transmission selection page: Select “Use the native Windows Secure Channel library” (use the native Windows secure channel library, compatible with the Windows system) and click 【Next】;

  19. 行尾符号转换配置页：选择 “Checkout Windows-style, commit Unix-style line endings”（Windows 系统推荐设置，避免换行符兼容问题），点击 【Next】；

  20. Line ending conversion configuration page: Select “Checkout Windows-style, commit Unix-style line endings” (recommended setting for Windows systems to avoid line break compatibility issues) and click 【Next】;

  21. 终端模拟器配置页：选择默认选项 “Use MinTTY (the default terminal of MSYS2)”（Git Bash 默认终端，体验更优），点击 【Next】；

  22. Terminal emulator configuration page: Select the default option “Use MinTTY (the default terminal of MSYS2)” (the default terminal for Git Bash with a better experience) and click 【Next】;

  23. 后续页面无需额外配置，均点击 【Next】 或 【Install】，等待安装完成后点击 【Finish】 即可。

  24. No additional configuration is required for subsequent pages; click 【Next】 or 【Install】 for all. After the installation is complete, click 【Finish】.

四、安装验证（可选，确保工具可用）

4. Installation Verification (Optional, to Ensure Tools Work)

安装完成后，可通过以下步骤验证工具是否正常运行：
After installation, you can verify whether the tools work normally through the following steps:

  1. 按下 Win + R，输入 cmd 打开命令提示符；

  2. Press Win + R, enter cmd, and open the Command Prompt;

  3. 输入 node -v，若显示 Node.js 版本号（如 v24.6.0），说明 Node.js 安装成功；

  4. Enter node -v; if the Node.js version number (e.g., v24.6.0) is displayed, it means Node.js is installed successfully;

  5. 输入 git --version，若显示 Git 版本号（如 git version 2.51.0.windows.1），说明 Git 安装成功。

  6. Enter git --version; if the Git version number (e.g., git version 2.51.0.windows.1) is displayed, it means Git is installed successfully.

至此，博客搭建的基础工具已全部准备完毕，下一步可开始安装 Hexo 框架并初始化博客。
At this point, all basic tools for blog building are ready. The next step is to install the Hexo framework and initialize the blog.
